# 标书核对工作台分页与性质筛选设计

日期：2026-06-25

> 分页条数后续已由 `2026-06-25-unified-ten-item-pagination-design.md` 统一调整为每页 10 条。

## 目标

将标书解析后的核对结果从固定高度滚动表格改为分页表格，减少内层滚动带来的浏览负担；同时将顶部核对状态筛选改为要求性质筛选，让用户优先按后续处理动作浏览要求。

## 页面交互

### 顶部筛选

顶部快捷筛选固定为三项：

- 全部
- 能力要求
- 提交资料

对应数据字段：

- 全部：不限制 `requirement_nature`
- 能力要求：`requirement_nature=capability`
- 提交资料：`requirement_nature=submission`

删除现有“全部性质”下拉框，避免重复筛选入口。

分类下拉和关键词搜索保留。无论是否已完成匹配，顶部快捷按钮始终使用性质分类；解析状态和匹配状态都在表格对应状态列中显示。

### 表格状态

“待确认 / 页码定位失败 / 已确认”不再作为顶部快捷筛选按钮，统一保留在表格“解析状态”列中。

状态显示规则不变：

- 无有效页码：定位失败
- 有页码且 `review_status=pending`：待确认
- 有页码且 `review_status=confirmed`：已确认

### 分页

- 默认每页 20 条。
- 使用 MUI 分页组件显示在表格底部。
- 表格容器取消固定高度、`overflow: auto` 和 sticky header，不再显示内部滚动条。
- 页码数量基于全部筛选条件处理后的结果计算。
- 关键词、分类、性质或匹配状态变化时自动回到第 1 页。
- 数据增删导致当前页越界时，自动回退到最后一个有效页。

### 选择与序号

- 表头全选仅作用于当前页。
- 已在其他页选中的条目保持选中状态。
- 批量确认和批量删除继续作用于所有已选条目。
- 表格序号按筛选结果连续计算，例如第 2 页第一条显示 21。
- 底部显示当前范围和总数，例如“21–40 / 共 70 项”。

### 新增与编辑

- 手动新增编辑行显示在当前表格顶部。
- 新增成功后回到第 1 页，确保用户能看到新条目。
- 编辑已有条目后保留当前页；如果修改使该条目不再符合当前筛选条件，则它从当前页消失，并重新校正页码。
