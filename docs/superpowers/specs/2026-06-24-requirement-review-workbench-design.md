# 标书解析核对工作台设计

## 目标

将标书要求详情从纵向折叠卡片改造成高密度的左右分栏核对工作台，支持用户快速完成“查看 AI 提取结果 → 对照原 PDF → 修正 → 确认”的闭环。

## 布局

- 顶部：当前文件、解析总数、分类统计、搜索和状态筛选。
- 左侧 58%：紧凑要求表格，首屏展示至少 15 条。
- 右侧 42%：原始 PDF 阅读器及当前要求原文片段。
- 底部：批量确认、批量删除和开始资质匹配。

## 左侧表格

- 列：复选框、序号、解析状态、类别、AI 提取结果、页码、轻量操作。
- 默认排序：页码定位失败 → 未确认 → 已确认。
- 点击行选择当前要求，右侧同步跳转。
- 支持键盘上下方向键切换选中行。
- 长内容最多显示两行；完整内容在编辑弹窗中展示。
- 支持单条修改、删除、确认。
- 支持批量确认和批量删除。

## 解析确认状态

- `pending`：未人工确认。
- `confirmed`：人工已确认。
- `location_failed`：由 `page_number` 为空动态判定，优先级最高，不单独持久化。
- 当前没有模型置信度字段，因此不展示虚构置信度。

## PDF 对照

- 后端提供当前标书 PDF 文件流。
- 右侧使用浏览器原生 PDF 阅读器，URL fragment 使用 `#page=N&zoom=page-width` 跳页。
- 顶部显示当前要求标题、页码和原始文本高亮块。
- 当前解析链没有文字坐标，无法在 PDF 页面内绘制精确高亮框；原文片段高亮作为当前可验证实现。
- 页码缺失时，PDF 保持当前页并显示定位失败提示。

## 编辑与删除

- 编辑弹窗允许修改分类、标题、完整内容、硬性属性、页码和数值规则。
- 删除单条要求需二次确认。
- 批量删除需二次确认。
- 所有修改立即写入 SQLite 并刷新当前列表。

## 数据契约

- `tender_requirements` 增加 `review_status TEXT NOT NULL DEFAULT 'pending'`。
- 要求 API 返回后端原始字段，并由前端统一映射为前端模型。
- 新增 PDF、更新、删除、批量确认、批量删除接口。

## 验收标准

- 96 条数据首屏可看到至少 15 条。
- 点击有页码的行，PDF 自动跳转到对应页。
- 页面不再出现重复的“完整要求”展示框。
- 定位失败和未确认项默认置顶。
- 行内修改、删除、批量确认和批量删除可用。
- TypeScript 构建、后端测试和浏览器交互检查通过。
