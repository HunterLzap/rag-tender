# 资质列表按分类动态表头设计

## 背景

资质库页的“资质列表”当前对所有分类都使用企业证书字段：名称、编号、发证日期、有效期、发证机构、认证范围、等级、持证主体。这个表头适合企业资质证书，但不适合财务、人员、业绩等资料。

## 目标

在不改数据库、不改后端接口、不影响上传解析流程的前提下，前端根据当前资质分类切换表格表头和单元格展示文案，让不同性质的资质资料显示为更贴近业务语义的表格。

## 范围

本次只修改资质库页“资质列表”的展示层：

- 保留现有 10 条/页分页。
- 分类筛选不再提供“全部分类”，进入资质列表时默认展示“企业资质”。
- 保留现有新增、编辑、删除能力。
- 保留后端 `Qualification` 字段结构：`name`、`number`、`issue_date`、`expiry_date`、`issuing_authority`、`scope`、`level`、`holder`。
- 不新增数据库字段。
- 不调整 LLM 解析 Prompt。
- 不调整文件上传页。

## 分类列定义

当分类筛选为单一分类时，表头按该分类展示：

| 分类 | 列定义 |
|---|---|
| 企业资质 | 名称、编号、发证日期、有效期至、发证机构、认证范围、等级、持证主体、操作 |
| 人员资质 | 人员/证书名称、证书编号、发证日期、有效期至、发证机构、专业/范围、等级、持有人、操作 |
| 业绩 | 项目/业绩名称、合同/项目编号、签订/开始日期、完成/有效期、业主/发包方、项目范围、金额/等级、主体、操作 |
| 财务 | 资料名称、年度/编号、出具日期、覆盖期/有效期、出具机构、指标/范围、结论/等级、主体、操作 |
| 其他 | 名称、编号、日期、有效期、来源/机构、内容摘要、等级/类型、主体、操作 |

不提供“全部分类”。资质列表始终处在一个明确分类内，避免不同性质资料混在同一张表里导致表头语义不统一。

## 数据映射

本次只换表头语义，不换底层字段：

| 显示含义 | 使用字段 |
|---|---|
| 名称/项目/资料/人员证书 | `name` |
| 编号/年度/合同编号 | `number` |
| 发证/签订/出具日期 | `issue_date` |
| 有效期/覆盖期/完成时间 | `expiry_date` |
| 发证机构/业主/出具机构/来源 | `issuing_authority` |
| 范围/摘要/指标 | `scope` |
| 等级/金额/结论 | `level` |
| 持证主体/持有人/主体 | `holder` |

日期字段继续复用现有格式化逻辑。空值显示 `—`；有效期为空时继续显示 `长期有效`。

## 测试与验收

- 新增前端纯函数测试，验证每个分类返回正确表头。
- 验证分类筛选没有“全部分类”，默认分类为企业资质。
- 验证资质列表分页仍为每页 10 条。
- 运行 TypeScript 构建。
- 在浏览器访问 `/knowledge`，切换资质列表的分类筛选，确认表头随分类变化且没有内部滚动条回退。

## 自检

- 无数据库迁移。
- 无接口字段变更。
- 无未定义占位项。
- 展示语义和当前后端字段一一对应。
