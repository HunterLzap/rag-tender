# 标书要求详情页信息层级优化设计

## 目标

将当前一次性铺开全部要求的长表格，改为“概览统计 + 快速筛选 + 分类折叠 + 条目展开”的详情视图，使用户能快速判断要求总量、硬性要求和数值门槛。

## 范围

- 仅调整前端标书解析详情视图。
- 不修改后端接口、数据库和数据模型。
- 保留淡紫色 `#7C4DFF`、白色背景、直角边框和无阴影风格。

## 页面结构

1. 详情标题栏
   - 左侧显示标书名称和元数据。
   - 右侧显示“开始匹配”主按钮。
   - 标题栏在详情区域内吸顶。

2. 概览统计
   - 全部要求数量。
   - 硬性要求数量。
   - 含数值条件数量。
   - 缺少页码、标题或内容的需关注数量。

3. 筛选工具栏
   - 分类标签：全部、资质、业绩、财务、人员、其他。
   - 关键词搜索标题和内容。
   - “只看硬性”开关。
   - “只看数值要求”开关。

4. 分类折叠区
   - 按资质、业绩、财务、人员、其他固定顺序展示。
   - 默认展开资质分类，其余折叠。
   - 分组标题显示分类名称、筛选后数量和硬性数量。
   - 筛选后无结果的分类不展示。

5. 要求条目
   - 摘要行显示硬性标签、标题、数值条件、页码和展开按钮。
   - 第二行最多展示两行内容摘要。
   - 展开后显示完整要求与原始文本。
   - 页码或关键文本缺失时显示“需核对”提示。

## 状态行为

- 切换分类、关键词或开关后，统计总览保持原始解析数据统计，分类分组显示筛选结果。
- 分类标签为“全部”时展示所有有结果的分组。
- 点击要求行只切换该要求的展开状态。
- 解析中、失败和空结果状态保持现有逻辑。

## 验收标准

- 100 条以上要求不再默认全部展开完整内容。
- 用户无需滚动到底部即可开始匹配。
- 用户可在一次点击内只查看硬性要求或数值要求。
- 用户可按分类和关键词定位要求。
- TypeScript 编译和生产构建通过。
